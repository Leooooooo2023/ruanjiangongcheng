# 小区物业管理系统 — 流程图（简化格式）

---

## 图 1：系统架构图

```
1-开始-系统启动
2-输入-浏览器访问管理后台(:5173) 或 业主端(:5174)
3-处理-Vue Router 路由守卫检查 localStorage token
4-判断-token存在且未过期?
  是-5
  否-6
5-处理-加载 AdminLayout / PortalLayout 布局组件
6-输出-跳转登录页(/login)
7-输入-前端Axios拦截器自动附加 Authorization: Bearer {token}
8-处理-请求到达Spring Boot后端(:8080)
9-处理-CORS跨域配置放行(localhost:*)
10-处理-JwtInterceptor拦截/api/**路径
11-判断-Token有效(非空+Bearer格式+签名验证)?
  是-12
  否-13
12-处理-解析Token,提取userId/username/role存入Request Attribute
13-输出-返回401 JSON → 前端清除token跳转登录
14-处理-RoleInterceptor角色权限校验
15-判断-role=0访问管理API 或 role=1访问业主API?
  是-16
  否-17
16-处理-路由至对应Controller(9个)
17-输出-返回403 JSON 无权访问
18-处理-Controller参数校验(@Validated)
19-判断-参数合法?
  是-20
  否-21
20-处理-调用Service业务逻辑层(8个)
21-输出-返回400 JSON 字段校验错误
22-处理-Service调用DAO/Mapper接口(7个)
23-处理-MyBatis-Plus执行SQL操作MySQL(7张表)
24-处理-返回Result.success(data)或Result.error(code,msg)
25-处理-Axios响应拦截器解析数据
26-判断-响应code=200?
  是-27
  否-28
27-处理-更新Pinia Store和页面数据
28-输出-ElMessage弹出错误提示
29-处理-文件上传专属路径:/api/files/upload → 保存至D:/property-uploads/
30-处理-虚拟路径/uploads/**映射至物理文件路径
31-结束-Vue页面渲染完成
```

## 图 2：数据库 E-R 图

```
1-开始-数据库设计
2-处理-admin管理员表
    字段:id(PK),username(UK),password(BCrypt),name,phone,email,is_deleted,create_time,login_fail_count,lock_until
    关系:独立表,不与其他表关联
3-处理-building楼栋表
    字段:id(PK),name,units,floors,description,is_deleted,create_time
    关系:1:N → owner表(building_id外键)
4-处理-owner业主表
    字段:id(PK),username(UK),password(BCrypt),name,phone,email,building_id(FK→building.id),unit,room,is_deleted,create_time,login_fail_count,lock_until
    关系:N:1 ← building表(building_id)
    关系:1:N → parking表(owner_id)
    关系:1:N → repair表(owner_id)
    关系:1:N → complaint表(owner_id)
5-处理-employee员工表
    字段:id(PK),name,phone,position,department,is_deleted,create_time
    关系:1:N → repair表(employee_id)
6-处理-parking停车位表
    字段:id(PK),number,location,status(0空闲/1占用),owner_id(FK→owner.id),start_date,end_date,create_time
    关系:N:1 ← owner表(owner_id)
7-处理-repair报修表
    字段:id(PK),owner_id(FK→owner.id),title,content,image,status(0待处理/1维修中/2已完成),employee_id(FK→employee.id),rating(1-5),create_time,update_time
    关系:N:1 ← owner表(owner_id)
    关系:N:1 ← employee表(employee_id)
8-处理-complaint投诉表
    字段:id(PK),owner_id(FK→owner.id),title,content,reply,status(0待回复/1已回复),create_time,update_time
    关系:N:1 ← owner表(owner_id)
9-结束-共7张表,5条外键关联,全部支持软删除(is_deleted)
```

## 图 3：用户认证流程图

```
1-开始-用户访问登录页面
2-输入-前端GET /api/auth/captcha加载验证码图片
3-处理-CaptchaUtil.generate()生成4位字符130x48 PNG
4-处理-验证码存入ConcurrentHashMap(key→图片,5分钟过期)
5-输入-用户填写表单:{username,password,role,验证码}
6-判断-前端字段完整性校验通过?
  是-7
  否-5
7-处理-POST /api/auth/login提交后端
8-判断-CaptchaUtil.verify(key,code)验证码正确?
  是-9
  否-10
9-处理-验证码一次性删除(防重复使用)
10-输出-返回400:验证码错误或已过期→5
11-判断-role=0(管理员)还是role=1(业主)?
  是-12(管理员)
  否-20(业主)
12-处理-adminMapper.selectOne(username)查询admin表
13-判断-管理员记录存在?
  是-14
  否-15
14-处理-检查lockUntil:锁定截止时间>当前时间?
15-输出-返回401:用户名或密码错误→5
16-判断-账户处于锁定状态?
  是-17
  否-18
17-输出-返回403:账户已锁定30分钟→5
18-处理-BcryptUtil.check(password,hashed)验证密码
19-处理-管理员分支跳至29
20-处理-ownerMapper.selectOne(username)查询owner表
21-判断-业主记录存在?
  是-22
  否-15
22-处理-检查lockUntil:锁定截止时间>当前时间?
23-判断-账户处于锁定状态?
  是-17
  否-24
24-处理-BcryptUtil.check(password,hashed)验证密码
25-判断-密码正确?
  是-26
  否-27
26-处理-resetLock():loginFailCount归零,lockUntil置null→30
27-处理-loginFailCount自增+1
28-判断-失败次数>=5?
  是-29
  否-30
29-处理-setLockUntil(now+30min):锁定30分钟
30-输出-返回403:密码错误已锁定30分钟→5
31-输出-返回401:还剩(5-N)次机会→5
32-处理-jwtUtil.generateToken(id,username,role):HMAC256签名,24h过期
33-输出-返回{token,id,username,name,role}
34-处理-前端localStorage存储token+Pinia更新状态
35-判断-路由守卫:role=0还是role=1?
  是-36(role=0)
  否-37(role=1)
36-输出-跳转管理后台/dashboard
37-输出-跳转业主端/home
38-结束-登录成功进入系统
```

## 图 4：业主注册流程图

```
1-开始-业主访问注册页面/register
2-输入-显示注册表单:用户名/密码/确认密码/姓名/手机/邮箱/楼栋/单元/房间
3-输入-用户填写所有字段
4-判断-前端校验:字段完整+两次密码一致?
  是-5
  否-3
5-处理-POST /api/auth/register提交RegisterDTO
6-处理-@Validated服务端自动校验(@Size/@NotBlank)
7-判断-服务端校验通过?
  是-8
  否-9
8-处理-adminMapper.selectCount(username)查admin表
9-输出-返回400:字段校验错误详情→3
10-处理-ownerMapper.selectCount(username)查owner表
11-判断-adminCount>0 或 ownerCount>0?
  是-12
  否-13
12-输出-返回BusinessException:用户名已存在→3
13-处理-BcryptUtil.encrypt(password)密码BCrypt加密
14-处理-构建Owner实体:setUsername/setPassword/setName/setPhone/setEmail/setBuildingId/setUnit/setRoom
15-处理-ownerMapper.insert(owner)插入数据库
16-处理-MyBatis-Plus自动填充createTime
17-输出-返回200:注册成功
18-处理-前端跳转登录页/login
19-结束-注册完成
```

## 图 5：报修工单生命周期流程图

```
1-开始-报修流程
2-输入-业主填写报修表单:标题+内容+图片(可选)
3-处理-POST /api/owner/repairs提交报修
4-处理-Service创建Repair实体:ownerId=当前业主,status=0(待处理),employeeId=NULL,rating=NULL
5-处理-repairMapper.insert()入库
6-输出-status=0 待处理:工单进入待分配队列

7-输入-管理员查看报修列表GET /api/repairs(分页+业主名+员工名)
8-处理-管理员选择工单和负责维修员
9-处理-PUT /api/repairs/{id}/assign:设置employeeId,status变更为1
10-处理-repairMapper.updateById()
11-输出-status=1 维修中:已指派维修员

12-输入-维修员上门维修,完成维修
13-处理-管理员PUT /api/repairs/{id}/status:status变更为2
14-处理-repairMapper.updateById()+自动更新updateTime
15-输出-status=2 已完成:等待业主评价

16-输入-业主查看已完成工单GET /api/owner/repairs
17-判断-工单status=2且rating=NULL?
  是-18
  否-19
18-处理-业主PUT /api/owner/repairs/{id}/rate提交评分(1-5星)
19-输出-工单已评价或不可评价
20-处理-Service校验:仅status=2可评价
21-处理-repairMapper.updateById()更新rating字段
22-输出-评分已完成

23-结束-报修流程结束(待处理→维修中→已完成→已评价)
```

## 图 6：投诉建议处理流程图

```

3-处理-POST /api/owner/complaints提交投诉
4-处理-Service创建Complaint实体:ownerId=当前业主,status=0(待回复),reply=NULL

9-输入-管理员填写回复内容
10-处理-PUT /api/complaints/{id}/reply:写入reply字段
11-处理-Service设置status=1(已回复),自动更新updateTime
12-处理-complaintMapper.updateById()
13-输出-status=1 已回复:回复完成

14-输入-业主查看我的投诉GET /api/owner/complaints
15-判断-投诉status=1(已回复)?
  是-16
  否-17
16-输出-显示管理员回复内容(reply字段)
17-输出-显示待回复状态

18-结束-投诉流程结束(待回复→已回复)
```

## 图 7：车位管理流程图

```
1-开始-车位管理流程
2-输入-管理员创建车位POST /api/parkings:{number,location}
3-处理-parkingMapper.insert():status默认0(空闲),ownerId=NULL
4-输出-status=0 空闲:车位可分配

5-输入-管理员选择空闲车位+业主+使用日期范围
6-处理-PUT /api/parkings/{id}/assign
7-处理-Service设置:ownerId=业主ID,startDate=开始日期,endDate=结束日期,status=1
8-处理-parkingMapper.updateById()
9-输出-status=1 已占用:车位已分配给业主

10-输入-业主查看车位GET /api/owner/parking(只读)
11-输出-展示业主自己的车位信息

12-输入-管理员撤销车位分配
13-处理-PUT /api/parkings/{id}/revoke
14-处理-Service使用UpdateWrapper显式设置:ownerId=NULL,startDate=NULL,endDate=NULL,status=0
15-处理-parkingMapper.update(null, updateWrapper)
16-输出-status=0 空闲:车位重新可分配

17-结束-车位状态在空闲(0)和占用(1)之间循环
```
