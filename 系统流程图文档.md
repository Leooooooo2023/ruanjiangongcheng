## 图 3：用户认证流程图

```mermaid
flowchart TD
    Start(["👤 用户访问登录页"]) --> LoadCaptcha["GET /api/auth/captcha<br/>加载验证码图片"]
    LoadCaptcha --> ShowForm["显示登录表单<br/>用户名 + 密码 + 验证码 + 角色"]

    ShowForm --> Submit["用户点击登录"]
    Submit --> ValidateClient{"前端校验<br/>字段完整性"}
    ValidateClient -->|"不完整"| ShowForm
    ValidateClient -->|"完整"| CallAPI["POST /api/auth/login<br/>{username, password, role, captchaKey, captchaCode}"]

    CallAPI --> CheckCaptcha{"CaptchaUtil.verify()<br/>验证码校验"}
    CheckCaptcha -->|"错误/过期"| CaptchaError["❌ 400: 验证码错误或已过期"]
    CaptchaError --> ShowForm

    CheckCaptcha -->|"通过(一次性,删除)"| RoleBranch{"判断 role 角色"}

    RoleBranch -->|"role=0 (管理员)"| FindAdmin["adminMapper.selectOne()<br/>按username查询admin表"]
    RoleBranch -->|"role=1 (业主)"| FindOwner["ownerMapper.selectOne()<br/>按username查询owner表"]

    FindAdmin --> CheckAdminExist{"管理员存在?"}
    CheckAdminExist -->|"否"| AuthFail["❌ 401: 用户名或密码错误"]
    FindOwner --> CheckOwnerExist{"业主存在?"}
    CheckOwnerExist -->|"否"| AuthFail

    CheckAdminExist -->|"是"| CheckLock{"检查锁定状态<br/>lockUntil > now?"}
    CheckOwnerExist -->|"是"| CheckLockOwner{"检查锁定状态<br/>lockUntil > now?"}

    CheckLock -->|"已锁定"| Locked["❌ 403: 账户已锁定<br/>请30分钟后再试"]
    CheckLockOwner -->|"已锁定"| Locked

    CheckLock -->|"未锁定"| VerifyPwd{"BcryptUtil.check()<br/>密码验证"}
    CheckLockOwner -->|"未锁定"| VerifyPwdOwner{"BcryptUtil.check()<br/>密码验证"}

    VerifyPwd -->|"错误"| IncFail["loginFailCount +1"]
    VerifyPwdOwner -->|"错误"| IncFailOwner["loginFailCount +1"]

    IncFail --> CheckCount{"失败次数 >= 5?"}
    IncFailOwner --> CheckCountOwner{"失败次数 >= 5?"}

    CheckCount -->|"是"| SetLock["setLockUntil(now+30min)<br/>❌ 403: 已锁定30分钟"]
    CheckCountOwner -->|"是"| SetLockOwner["setLockUntil(now+30min)<br/>❌ 403: 已锁定30分钟"]

    CheckCount -->|"否"| RemainAttempt["❌ 401: 还剩N次机会"]
    CheckCountOwner -->|"否"| RemainAttemptOwner["❌ 401: 还剩N次机会"]

    SetLock --> ShowForm
    SetLockOwner --> ShowForm
    RemainAttempt --> ShowForm
    RemainAttemptOwner --> ShowForm
    AuthFail --> ShowForm
    Locked --> ShowForm

    VerifyPwd -->|"正确"| ResetLock["resetLock()<br/>failCount=0, lockUntil=null"]
    VerifyPwdOwner -->|"正确"| ResetLockOwner["resetLock()<br/>failCount=0, lockUntil=null"]

    ResetLock --> GenToken["jwtUtil.generateToken()<br/>payload: {id, username, role}<br/>有效期: 24小时"]
    ResetLockOwner --> GenToken

    GenToken --> ReturnResult["✅ 返回 {token, id, username, name, role}"]
    ReturnResult --> FrontStore["前端存储到 localStorage<br/>+ Pinia Store"]
    FrontStore --> RouteRedirect{"路由守卫判断"}

    RouteRedirect -->|"role=0"| AdminDash["跳转 /dashboard<br/>管理后台首页"]
    RouteRedirect -->|"role=1"| PortalHome["跳转 /home<br/>业主端首页"]

    style Start fill:#e3f2fd,stroke:#1565c0
    style AdminDash fill:#c8e6c9,stroke:#2e7d32
    style PortalHome fill:#c8e6c9,stroke:#2e7d32
    style CaptchaError fill:#ffcdd2,stroke:#c62828
    style AuthFail fill:#ffcdd2,stroke:#c62828
    style Locked fill:#ffcdd2,stroke:#c62828
    style SetLock fill:#ffcdd2,stroke:#c62828
    style RemainAttempt fill:#fff9c4,stroke:#f57f17
    style ReturnResult fill:#c8e6c9,stroke:#2e7d32
```

**关键安全机制：**
1. **验证码**：Java AWT 生成 4 位字符图片，存入 `ConcurrentHashMap`，5 分钟过期，一次性使用
2. **密码加密**：BCrypt 哈希，不存储明文
3. **登录锁定**：连续输错 5 次 → 锁定 30 分钟；成功后重置计数
4. **JWT Token**：Auth0 java-jwt 4.x，HMAC256 签名，24 小时过期

## 图 4：业主注册流程图

```mermaid
flowchart TD
    Start(["🏠 业主访问注册页"]) --> ShowReg["显示注册表单<br/>用户名/密码/确认密码/姓名<br/>手机/邮箱/楼栋/单元/房间"]

    ShowReg --> FillForm["用户填写表单"]

    SubmitReg --> ClientCheck{"前端校验"}

    ClientCheck -->|"格式正确"| CallAPI["POST /api/auth/register<br/>RegisterDTO<br/>@Validated 自动校验"]

    CallAPI --> ServerValid{"服务端 @Valid 校验"}
    ServerValid -->|"校验失败"| ValidErr["❌ 400: 字段校验错误<br/>(@Size, @NotBlank等)"]
    ValidErr --> ShowReg

    ServerValid -->|"通过"| CheckAdmin["adminMapper.selectCount()<br/>检查admin表用户名"]

    DupCheck -->|"是"| DupErr["❌ BusinessException<br/>用户名已存在"]
    DupErr --> ShowReg

    DupCheck -->|"否"| EncryptPwd["BcryptUtil.encrypt(password)<br/>密码BCrypt加密"]

    EncryptPwd --> BuildEntity["构建 Owner 实体<br/>setUsername/setPassword/setName<br/>setPhone/setEmail<br/>setBuildingId/setUnit/setRoom"]

    BuildEntity --> InsertDB["ownerMapper.insert(owner)<br/>MyBatis-Plus 自动填充 createTime"]

    InsertDB --> Success(["✅ 注册成功"])
    Success --> Redirect["跳转登录页<br/>提示: 注册成功,请登录"]

    style Start fill:#e3f2fd,stroke:#1565c0
    style Success fill:#c8e6c9,stroke:#2e7d32
    style DupErr fill:#ffcdd2,stroke:#c62828
    style ValidErr fill:#ffcdd2,stroke:#c62828
```

**关键设计点：**
- 用户名唯一性检查 **跨 admin 和 owner 两张表**，防止业主注册与管理员同名
- 默认密码未显式设置（代码中 `owner123` 仅种子数据使用），实际注册密码由用户填写
- 使用 `@Validated` + Jakarta Validation 注解做服务端二次校验

## 图 7：车位管理流程图

```mermaid
flowchart TD

    S0 --> A_Assign["🔧 管理员分配车位<br/>PUT /api/parkings/{id}/assign<br/>选择业主 + 设置日期范围"]
    A_Assign --> S1["status=1<br/>🚗 已占用"]

    S1 --> A_Revoke["🔧 管理员撤销分配<br/>PUT /api/parkings/{id}/revoke<br/>清空ownerId和日期"]
    A_Revoke --> S0

    O_View["👤 业主查看车位<br/>GET /api/owner/parking<br/>只读查看自己的车位信息"]

    style S0 fill:#c8e6c9,stroke:#2e7d32,color:#000
    style S1 fill:#ffcdd2,stroke:#c62828,color:#000

**核心操作：**

| 操作 | API | 说明 |
|------|-----|------|
| 新增车位 | `POST /api/parkings` | 管理员创建，默认 status=0(空闲) |
| 分配车位 | `PUT /api/parkings/{id}/assign` | 绑定业主 + 设置 startDate/endDate，status→1 |
| 撤销分配 | `PUT /api/parkings/{id}/revoke` | 清空 ownerId/startDate/endDate，status→0 |
| 查看车位 | `GET /api/owner/parking` | 业主端只读查看自己的车位 |

**技术细节：** 撤销操作使用 MyBatis-Plus `UpdateWrapper` 显式设置 null 值，因为默认策略会忽略 null 字段。

sequenceDiagram
    actor User as 👤 用户
    participant FE as 🖥️ 前端 (Vue 3)
    participant Axios as 📡 Axios 拦截器
    participant CORS as 🔧 CORS Filter
    participant JWT as 🔐 JwtInterceptor
    participant Role as 🛡️ RoleInterceptor
    participant Ctrl as 🎮 Controller
    participant Svc as 📋 Service
    participant DAO as 🗄️ Mapper (MyBatis-Plus)
    participant DB as 💾 MySQL

    User->>FE: 操作页面 (点击/提交表单)
    FE->>Axios: 发起请求

    alt 请求需要认证
        Axios->>Axios: 请求拦截器<br/>从 localStorage 读取 token<br/>设置 Authorization: Bearer {token}
    end

    Axios->>CORS: HTTP 请求 (GET/POST/PUT/DELETE)
    CORS->>CORS: 校验 Origin<br/>放行 localhost:*
    CORS->>JWT: 进入拦截器链

    alt OPTIONS 预检
        JWT-->>Axios: 直接放行 200
    end

    JWT->>JWT: 从 Header 提取 Token<br/>去掉 "Bearer " 前缀
    alt Token 缺失/无效/过期
        JWT-->>Axios: ❌ 401 JSON<br/>"未登录或Token缺失"
        Axios->>Axios: 响应拦截器<br/>清除 token, 跳转登录页
    end

    JWT->>JWT: jwtUtil.validate(token)<br/>解析并验证签名
    JWT->>JWT: 设置 request attributes<br/>userId, username, role
    JWT->>Role: 放行

    Role->>Role: 读取 request.getAttribute("role")
    Role->>Role: 判断 URI 路径

    alt /api/owner/** 且 role≠1
        Role-->>Axios: ❌ 403 JSON<br/>"无权访问业主功能"
    else /api/buildings... 且 role≠0
        Role-->>Axios: ❌ 403 JSON<br/>"无权访问管理员功能"
    else 权限匹配
        Role->>Ctrl: 放行
    end

    Ctrl->>Ctrl: 参数校验 @Validated
    alt 校验失败
        Ctrl-->>Axios: ❌ 400 JSON (字段错误)
    end

    Ctrl->>Svc: 调用业务方法
    Svc->>Svc: 业务逻辑处理<br/>(唯一性检查/状态判断/密码加密等)

    alt 业务异常
        Svc-->>Ctrl: throw BusinessException
        Ctrl-->>Axios: ❌ 全局异常处理<br/>Result.error(code, msg)
    end

    Svc->>DAO: Mapper 方法调用
    DAO->>DB: SQL 执行
    DB-->>DAO: 查询结果
    DAO-->>Svc: 实体/DTO
    Svc-->>Ctrl: 业务结果
    Ctrl-->>Axios: ✅ Result.success(data)<br/>200 JSON

    Axios->>Axios: 响应拦截器<br/>提取 response.data
    alt code !== 200
        Axios->>Axios: ElMessage.error(message)
    end

    Axios-->>FE: 返回数据
    FE->>FE: 更新页面数据/UI
    FE-->>User: 显示结果
```
